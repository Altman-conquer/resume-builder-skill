# Template Selection Guide

Use this guide after reading `templates.json`.

## Choose by Output Format

- Choose LaTeX when the user wants a polished PDF, tight one-page layout, strong typography, or an academic-looking technical resume.
- Choose Typst when the user wants a PDF-first workflow but prefers simpler syntax and faster iteration than LaTeX.
- Choose Markdown or Pandoc when the user values content editing, version control diffs, and multiple export targets.
- Choose HTML/JS when the user wants a web resume, portfolio page, or visual front-end presentation.
- Choose JSON Resume when structured resume data should be reused across themes.

## Choose by Candidate Profile

- New-grad or internship: prefer clear one-page templates with education, project, internship, competition, and skills sections.
- Experienced engineer: prefer dense templates that emphasize impact, systems, ownership, metrics, and selected projects.
- Research or graduate-school profile: prefer LaTeX or Typst templates with publication-friendly typography and room for papers.
- Front-end or design-oriented profile: consider HTML/JS templates only when the web presentation itself supports the user's goal.

## Recommendation Rules

Shortlist 2-3 templates and explain trade-offs in practical terms:

- build complexity;
- Chinese font support;
- content density;
- visual style;
- likely editing cost;
- license and attribution requirements.

Do not pick solely by screenshot quality. Favor templates that can be compiled, maintained, and legally reused.
